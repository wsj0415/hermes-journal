## 核心理念

### 名字的重要性

引用 David Placek（Lexicon Branding 创始人，Swiffer/BlackBerry/Pentium/Impossible Foods 幕后）:

> **"你品牌中所做的任何事情，都不会比你的名字使用得更频繁或更长久。"**

**名字是**:
- 人们听到的第一件事
- 最后忘记的事
- 出现在每张发票、每个邮件签名、每个 pitch deck、每次 Google 搜索
- 随时间复合

**好名字 vs 平庸名字**:
| 好名字 | 平庸名字 |
|--------|----------|
| 建立累积优势 | 每天产生摩擦 |
| 在每次互动中默默为你工作 | 每次都被注意到是错的 |

**现实**: 大多数创始人花更多时间选晚餐餐厅而不是给公司命名。

---

### 名字 vs 品牌

> **名字不是品牌。**

| 概念 | 定义 |
|------|------|
| **品牌** | 整体体验，人们心中的感知 |
| **名字** | 所有东西的把手，容器 |

**案例**: 高端艺术品市场客户原名"Nerd Crawler"→投资者不投→改名"**Rare Marq**"
- Rare: 独特物品
- Marq: 艺术家在实体作品上的签名标记
- 结果：增长非常好

**案例**: "Phoenix Air Charter"→"**Axio**"
- 货运喷气机世界中，标准运营方法
- 雄心使命：在最挑战条件下成为可依赖伙伴

---

## 好名字的三个特征

| 特征 | 说明 | 失败点 |
|------|------|--------|
| **1. 吸引注意** | 在噪音海洋中脱颖而出 | — |
| **2. 保持注意** | 易处理、易说、易记（语言学家称"处理流畅性"） | — |
| **3. 惊喜** | 有意外的东西，让你停顿半秒 | **大多数人失败在这里** |

---

### 隐形区问题

**大多数人选择**:
- 安全
- 描述他们做什么
- 结果：隐形

**案例**: 开云公司命名"CloudTech Solutions"
- 告诉了人们你做什么
- **也保证了他们永远不会记住你**

**隐形区特征**:
- 足够熟悉感觉舒适
- 足够有特色但…什么都不是
- 每个人点头"yeah, that's fine"

---

## 张力区 vs 隐形区（David Placek 框架）

### 隐形区

```
"每个人点头：'yeah, that's fine'"
"描述你做什么"
"感觉'专业'"
"没人讨厌它"
"结果：隐形，易忘"
```

**例子**:
- "CloudTech Solutions"
- "ProMop"（P&G 内部建议）
- "Zeit"（Vercel 原名）

---

### 张力区 ⭐ 推荐

```
"一半爱，一半恨"
"能量 HIGH"
"强烈反应"
"创造'wait, what?'时刻"
"结果：难忘，有特色"
```

**例子**:
- **"Pentium"**（Intel）— 内部两极分化
- **"Impossible Foods"** — 对抗性声明
- **"Swiffer"** — 俏皮、活跃
- **"BlackBerry"** — 科技产品的温暖有机感

---

### 核心洞察

> **"如果名字让房间里每个人都舒适，可能不够有特色。"**

> **"最好的名字创造一点张力。一点'wait, what?'。那能量是让人们记住你的东西。"**

**Intel Pentium 案例**:
- 工程师想要："ProChip" / "i586"（安全、技术、描述性）
- Lexicon 提议："Pentium"
- 内部真正反对
- CEO Andy Grove 决定："这是好名字因为它是两极分化的。那意味着它有能量。"
- 结果：历史上最知名的处理器名之一
- **同时期 AMD 芯片**: 不说名字需要 Google

**教训**:
> **如果名字让房间里每个人都舒适，可能不够有特色。**

## 三阶段命名流程

### Phase 1: 理解领域（命名前）

**核心决策**: **决定你不是什么** — 整个过程中最重要的决策

**行动项**:
```
1. 列出前 20 个竞争对手和他们的名字
2. 识别行业命名模式（描述性/发明/隐喻/首字母缩写/创始人名）
3. 决定哪些模式过度使用且禁止
4. 注意你欣赏的名字及原因
```

**约束是解放的**:
- 消除明显的
- 强迫进入更有创意领域
- "我们知道不去那里"

---

### Phase 2: 利益阶梯（命名什么）

**大多数人错误**: 用公司做什么命名而非**意味着什么**

**Lexicon 纤维产品案例**:
- 明显领域：肠道健康、消化、纤维
- Lexicon 持续推动：更好消化的终极好处是什么？消费者实际感觉如何？
- 答案：**感觉更轻**
- 不是"healthy gut"或"daily fiber" — **Lighter**
- 重新框架打开全新创意领域
- 从医疗/健康语言到轻盈、空气、自由、运动相关词

**利益阶梯**:
```
1. 产品/服务实际做什么？（功能层）
        ↓
2. 那解决什么问题？（功能利益）
        ↓
3. 解决那问题让人感觉如何？（情感利益）
        ↓
4. 连接到什么更大抱负或身份？（终极利益）← 好名字住在这里
```

**案例：项目管理工具**:
| 层级 | 内容 |
|------|------|
| 功能 | 任务追踪和团队协作 |
| 功能利益 | 项目按时完成 |
| 情感利益 | 感觉受控，更少混乱 |
| **终极利益** | **清晰。一切处理好的感觉，可专注于重要的事。** |

**"Clarity"概念比"project management"给 100 倍更多创意燃料**。

**行动项**:
```
1. 为公司写出利益阶梯
2. 采访 5-10 个客户。问使用产品后感觉如何，非做什么
3. 用 1-2 词识别终极利益
4. 用那概念作为创意跳板
```

---

### Phase 3: 寻宝（生成名字）

**核心原则**: **数量导致质量**

**现实**:
- 大多数创始人头脑风暴 50-100 个名字，选最不难听的，完成
- **Lexicon 每项目生成超 2000 个候选**

**你不需要 2000，但需要远超你认为的**。

---

#### 如何运行寻宝

**从词探索开始**:
取终极利益概念，从每个角度探索。如"clarity"：

| 角度 | 示例 |
|------|------|
| **同义词** | lucid, vivid, clear, transparent, sharp |
| **拉丁/希腊词根** | clar-, luc-, vid- |
| **相邻领域** | 光学（lens, prism, focus）/水（crystal, pure, spring）/天气（clear sky, horizon） |
| **声音和片段** | 什么音节感觉"清晰"？清脆辅音？开放元音？ |
| **隐喻** | 什么物体/体验代表清晰？铃响。山景。第一道光。 |

---

#### 近似思维（Approximate Thinking）

Placek 概念：

```
完全 bizarre/absurd/illegal ←──────→ 安全/可行/明显
                    ↑
            近似思维区
      未完全成型/未抛光/第一眼可能坏
              魔法住在这里
```

**给自己 permission 坐在那里**:
- 不要太早判断想法
- 写下奇怪的/半成型的/让你笑的
- 可稍后精炼，现在是狩猎

---

#### 声音象征（Sound Symbolism）

Lexicon 对声音象征痴迷：

| 声音 | 感觉 |
|------|------|
| **硬辅音（K, T, P）** | 强大、精确 |
| **软声音（L, M, S）** | 平滑、温柔 |
| **字母 V** | "英语语言中最大胆的字母"（Placek） |

**Vercel 案例**:
- 原名：Zeit（德语"时间"）
- 问题：没人会拼写，SEO 糟糕，名字不传达做什么
- Lexicon 改名：**Vercel**
- 源自："versatile" / "accelerate" / "excel"
- **前面 V 添加自信和能量**
- 公司现估值：**$93 亿**
- 名字没单独做到那，显然。但停止是负债开始是资产。

---

#### 行动项

```
1. 设定目标 300+ 名字候选（真的）
2. 花时间词源词典/同义词典/外语资源
3. 从至少 5 个相邻领域探索概念
4. 大声说每个名字。嘴里感觉如何？
5. 用 AI 生成 200-300 名字批次。然后应用人类判断。
```

## 5 个经典命名案例

### 1. Swiffer（P&G 清洁产品）

| 维度 | 内容 |
|------|------|
| **客户** | Procter & Gamble |
| **内部建议** | "ProMop" |
| **Lexicon 回应** | "这不是拖把。让我们放趣味进去。" |
| **创造** | **"Swiffer"** — 唤起迅速轻松扫掠和擦拭 |
| **特点** | 俏皮、活跃、听起来像使用产品动作 |
| **结果** | **$50 亿品牌** |
| **竞争对手** | Clorox "ReadyMop" — 安全、描述性、易忘，做了几亿 |

**教训**: 名字不仅反映产品，**重新框架整个品类**。

---

### 2. BlackBerry（RIM 邮件设备）

| 维度 | 内容 |
|------|------|
| **客户** | Research In Motion (RIM) |
| **风险** | "always-on email"在 2000 年代初听起来像噩梦 |
| **Lexicon 方法** | 不可预测 |
| **创造** | **"BlackBerry"** — 与邮件/科技无关，温暖、有机、亲切 |
| **微妙天才** | 黑莓上的小核果看起来像设备键盘上的小键 |
| **结果** | 让潜在恐吓科技感觉友好 |

---

### 3. Impossible Foods（植物基肉）

| 维度 | 内容 |
|------|------|
| **客户原始想要** | "crunchy and hippie"感觉适合 Whole Foods |
| **Lexicon 强烈反对** | "你不是做小众健康产品。你在做大胆声明植物基肉可比真实更好。" |
| **创造** | **"Impossible"** — 挑战、对抗、做声明 |
| **结果** | 强迫人们有意见 = 颠覆性品牌需要的 |

---

### 4. Pentium（Intel 处理器）

| 维度 | 内容 |
|------|------|
| **工程师想要** | "ProChip" / "i586"（安全、技术、描述性） |
| **Lexicon 提议** | **"Pentium"** — pent（五，第五代处理器）+ 听起来像化学元素的后缀，基础且强大 |
| **内部** | 真正反对 |
| **CEO Andy Grove 决定** | "这是好名字因为它是两极分化的。那意味着它有能量。" |
| **结果** | 历史上最知名处理器名之一 |
| **同时期 AMD** | 不说名字需要 Google |

---

### 5. Vercel（开发者工具）

| 维度 | 内容 |
|------|------|
| **原名** | Zeit（德语"时间"） |
| **问题** | 小开发者工具时 fine，但公司增长后名字成负债：难拼写/难搜索/无情感共鸣 |
| **改名** | **"Vercel"** — 干净、现代、有能量 |
| **来源** | "versatile" / "accelerate" / "excel" |
| **V 的力量** | "英语语言中最大胆的字母" |

## 完整 5 阶段命名练习

### Phase 1: 设置基础（1-2 小时）

```
1. 写出利益阶梯（功能→功能利益→情感利益→终极利益）
2. 列出 20 个竞争对手和名字。识别要避免的模式。
3. 定义"我们不去那里"边界。
4. 写下 3-5 个形容词描述听到名字时想让人感觉什么。
```

---

### Phase 2: 寻宝（2-3 小时）

```
1. 取终极利益概念，跨 5+ 相邻领域探索
2. 用词源词典（etymonline.com 免费）探索词根
3. 打开同义词典深入每个同义词
4. 尝试组合词片段。取一个词前半 + 另一个词后半。
5. 用 AI（ChatGPT/Claude）生成 200+ 候选。给概念/形容词/约束。
   要求：发明词/拉丁启发词/复合词/隐喻词。
6. 写下一切。先无过滤。目标最少 300 候选。
```

---

### Phase 3: 第一过滤（1 小时）

```
1. 大声读每个名字。划掉任何难说的。
2. 检查基础域名可用性（.com 没以前重要但仍检查）
3. 移除听起来像竞争对手的
4. 移除字面描述你做什么的
5. 应有 40-60 个幸存者
```

---

### Phase 4: 张力测试（1-2 小时）

```
1. 与 5-10 个信任的人分享前 40-60（非仅创始人，包括潜在客户）
2. 问："这名字让你感觉什么？"非"你喜欢吗？"感觉，非意见。
3. 注意两极分化的。如果一半爱一半恨 — 是信号。不要驳回。
4. 缩小到前 10。
```

---

### Phase 5: 最终评估（1 小时）

**给前 10 个打分（1-10）**:

| 标准 | 说明 |
|------|------|
| **独特性** | 从竞争对手中脱颖而出？ |
| **处理流畅性** | 易说、拼写、记住？ |
| **情感共鸣** | 连接到终极利益？ |
| **能量** | 创造反应？有张力？ |
| **长寿** | 10 年后仍有效？框住你？ |
| **可用性** | 域名（.com/.co/.io 或修改 get[name].com/[name]hq.com 等）和社交账号？用 Namechk/Instant Domain Search。别忘了商标（没人喜欢停止和终止信）。 |

**获胜者可能不是平均分最高的**。

**将是独特性和能量得 9 或 10 的那个，即使其他是 6**。

> **卓越胜过平衡。**

## AI 在命名中的角色

### AI 改变了命名格局

**现实**:
- ChatGPT/Claude 可几分钟生成 200-300 名字候选
- **不是夸张**。给好 brief 产生比大多数代理机构一周更多原始材料

**这意味着**:
> **竞争优势从生成转向判断。**

任何人都可创建长名单。难的部分、有价值的部分是**知道哪个名字对**。非仅好。**对**。

---

### AI 擅长 vs 不擅长

| AI 擅长 | AI 不擅长 |
|---------|----------|
| 数量 | 感觉 |
| 生成数百候选 | 告诉哪个名字有能量 |
| 快速产出 | 感知房间里呈现名字时的张力 |
| — | 感觉"意外熟悉"和"通用愉快"的差别 |

---

### 如何使用 AI

```
1. 用 AI 作为头脑风暴伙伴
2. 让它生成数百候选
3. 然后应用人类判断
4. 大声说名字
5. 在真人身上测试
6. 信任创造反应的直觉
```

**AI 注意事项**:
- 第一遍相当平淡通用
- **必须先做前面阶段** 才有方向工作
- 剧本没变。工具只是更快。

## 何时应该改名

### 创始人恐惧

> **"我们会失去建立的资产。"**

**现实**（Placek 说 Lexicon 从未在市场看到证据）:
- 如果年轻公司、Pre-Series B，名字资产最小
- 客户知道产品，非名字
- **带着热情和清晰故事完成的改名实际产生注意和能量**
- 是 PR 时刻。是联系每个客户告诉他们激动事的理由。

**真正风险**:
> **不是失去资产。是接下来十年用创造摩擦而非复合优势的名字。**

---

### 何时改名有意义

| 情况 | 建议 |
|------|------|
| **年轻公司/成长阶段** | 改名，现在做，便宜且容易 |
| **家喻户晓几十年认知** | 不同计算（Coca-Cola 不会改名） |

---

### 名字可能需要改变的信号

- [ ] 没人能正确拼写或发音
- [ ] 不断必须解释什么意思
- [ ] 描述你过去做什么，非成为什么
- [ ] 域名情况是灾难
- [ ] 人们把你和竞争对手混淆
- [ ] 大声说时有点畏缩

## 常见问题解答

### .com 域名多重要？

**比十年前不重要，但对 B2B 公司仍重要**。

- 客户会本能输入 name.com
- 如果不能获得确切.com，考虑.co/.io 或加词（get[name].com/[name]hq.com 等）
- **但不要让域名可用性杀死好名字**
- 域名可购买。**平庸名字是永远的。**

---

### 名字应该描述我们做什么吗？

**几乎总是不要。**

- 描述性名字感觉安全但框住你且让你隐形
- "General Electric"有效因为有一个世纪主导，非名字好
- **对初创公司，描述性名字是错失机会**

---

### 名字应该多长？

**1-3 音节是甜蜜点。2 音节理想。**

- 每额外音节降低记忆性
- 如果人们在对话中自然缩短名字 — 可能太长

---

### 发明词怎么样？

**历史上一些最好品牌名是发明的**: Kodak / Xerox / Spotify / Swiffer

**关键**: 需要感觉像可以是真实词
- 应遵循英语（或目标语言）语音规则
- "Xzqft" 是发明的但不是名字
- **"Spotify" 是发明的且感觉属于**

---

### 应该用焦点小组测试名字吗？

**测试，但非传统焦点小组**。

- 焦点小组优化共识，推你向隐形区
- **改为与 10-15 人一对一分享**
- 问"这让你感觉什么？"非"你喜欢这个吗？"
- 追踪哪个名字产生最强反应（正面或负面）

---

### B2B 公司命名同样重要吗？

**是。可能更重要。**

- B2B 买家是人类
- 他们在会议中谈论你、推荐给同事、演示前 Google 你
- **有特色、难忘的名字让每次接触点更容易**
- "B2B 可承受无聊"是商业中最昂贵神话之一

---

### 如果联合创始人对名字不同意？

**好。不同意意味着你在张力区。**

- 不要通过妥协向安全东西解决
- **改为与公司外人测试两个选项**
- 让数据（反应，非意见）打破平局

---

### 可以以创始人名字命名公司吗？

**可以，但仔细想为什么。**

- 创始人名字适用于某些类别（律所/咨询/奢侈品牌）— 个人声誉是产品
- **对科技/SaaS/或任何最终可能比你大的东西，独立名字给更多灵活性**

---

## 核心原则

> **你的名字是长期资产。它每天复合。**

> **值得花一个周末（或一个月，或雇人）做对。**

> **不要 settle 安全。不要描述你做什么。不要优化共识。**

> **找到有能量的名字。意外熟悉的那个。一半房间爱一半房间质疑的那个。**

> **那张力是功能，非 bug。**

> **如果当前名字不工作？改变它。最好时间是启动前。第二好时间是现在。**

## Lunour 命名 Skill 使用指南

### 如何加载技能到 Claude

```
1. 去 claude.com 打开设置
2. 导航到"Skills"点击"Add Skill"
3. 上传下载的 lunour-naming.skill 文件
4. 技能现在可用于任何对话
```

---

### 最佳实践

**从上下文开始，非请求**:
```
不要说："give me name ideas for my startup"
打开：关于公司的 brief
```

**强开场**:
> "I'm building a [what it does] for [who it's for]. The feeling I want people to have is [emotion]. Help me name it."

**完整回答发现问题**:
- 当 Claude 问品牌个性/目标客户/爱或恨的名字时，不要匆忙过
- 那些答案分离通用名单和实际适合的名字
- **"我恨的名字及为什么"问题特别强大**

**按类别要求名字**:
- "give me evocative names"
- "give me invented words only"
- "show me names that could become verbs"
- 技能有 10 种命名类型分类可工作

**用评估框架**:
- 有候选后，要求 Claude 给前 3-5 个对 7 个标准打分（memorable/spellable/ownable 等）
- 将直觉决定转为可辩护的结构化决定

**走到决赛选手开发**:
- 大多数人停在名单
- 技能可更远：为每个决赛选手写品牌起源故事/建议标语/映射域名替代/标记国际风险
- 明确要求："Now do a full finalist analysis on these three."

**那一行激活完整四阶段流程**而非直接跳到名字转储。
