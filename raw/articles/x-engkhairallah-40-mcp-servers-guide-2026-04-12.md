## 什么是 MCP

> **MCP 是最重要但没人讨论的 AI 基础设施。**

**核心对比**:
| Skills | MCP |
|--------|-----|
| 教 Claude **如何**做事 | 给 Claude **访问权**做事 |
| 内部能力 | 外部连接 |

---

### 没有 MCP 的 Claude

> **没有 MCP，Claude 是罐中大脑。** 它能思考/推理/生成文本。但它不能：
- 访问你的文件
- 搜索网页
- 查询数据库
- 检查邮件
- 读取日历
- 与任何外部系统交互

---

### 有 MCP 的 Claude

> **有 MCP，Claude 成为操作员。** 它能：
- 从任何地方拉取数据
- 推送结果到各处
- 在真实系统中执行真实行动

---

### MCP 定义

> **MCP = Model Context Protocol**  
> 开放标准，为 AI 连接外部工具和数据源创建一种通用方式。**一次构建连接，任何支持 MCP 的 AI 模型都能使用。**

## 安装流程

> **几乎所有服务器的流程相同：**

```
1. 克隆或安装服务器（通常 npm install 或 pip install）
2. 添加服务器配置到 Claude 设置
3. 提供任何需要的 API keys 作为环境变量
4. 重启 Claude
```

**大多数服务器设置耗时<5 分钟**。官方 Anthropic 服务器最简单 — 通常一条命令。

## 对 KilroyContentBot 的启示

### 可借鉴点
1. **MCP 连接器** — 给 AI 访问外部系统能力
2. **按角色推荐** — 不同用户类型不同组合
3. **官方服务器优先** — 最简单可靠
4. **fastmcp 自定义** — 无合适时自建
5. **MCPHub 管理** — 5+ 服务器时必要
6. **5 分钟设置** — 低摩擦采用

### 可实施
- 集成 Filesystem MCP 访问本地文件
- 集成 Context7 获取最新 API 文档
- 集成 Google Drive/Gmail/Calendar 连接器
- 集成 Obsidian MCP 访问知识库
- 用 fastmcp 构建自定义内容创作工具
- 用 MCPHub 管理多服务器
