## 🛠️ OpenClaw 是什么

**OpenClaw 是一个开源个人 AI 助手**，比我试过的任何东西都更强大、更自主、更有趣。

### 核心特性

| 特性 | 说明 |
|------|------|
| **消息平台** | 在你已经使用的平台上轻松消息它：WhatsApp、Telegram、Slack 等 |
| **电脑控制** | 可以控制你的电脑，像你自己一样完成任务 |
| **自主工作** | 按照自己的时间表工作，甚至通宵 |
| **始终在线** | 本地运行，可以构建自己的新技能 |
| **一次教学** | 教一次，它处理其余 |

### 实际示例

我可以给我的 OpenClaw 发这样的消息：

> "让我们确保我们的网站始终有最新的理由说明我们为什么比竞争对手更好。"

**它会：**
1. 使用网络搜索、我们的 GitHub 仓库和公共 API 找到所需信息
2. 提交 PR 与更新
3. 每周根据新功能或市场新闻刷新这些页面

**一条短信变成一个始终在线的代理。**

---

## 🔑 关键 OpenClaw 概念

| 概念 | 说明 |
|------|------|
| **本地网关** | 接收消息的通用收件箱，可以从任何渠道接收指令（终端、Telegram、WhatsApp 等） |
| **代理** | 在网关后面，有自己的身份、工具和工作空间 |
| **Cron 作业 + 心跳** | 代理按照预定的 cron 作业工作，每 30 分钟检查一次心跳 |
| **技能和工具** | 可以使用（并自安装！）技能、API 和 CLI 与系统和外部世界交互 |
| **部署** | 部署在拥有的机器上，如 Mac Mini 或云端的 VPS |

## 💻 安装选项

### ⚠️ 重要警告

**不要** 在积极使用的工作或个人电脑上安装 OpenClaw。这非常危险。

OpenClaw  technically 可以访问它运行电脑上所有文件，无论多么小心，你都不想冒删除一切或将个人文件发送给不可信角色的风险。

OpenClaw 团队在加强安全方面做得很好，但最好从隔离的盒子开始。

---

### 三个安全选项

#### 选项 1：托管版本 ⭐ 最简单

**供应商：**
- [StartClaw](https://startclaw.com/)
- [MyClaw](https://myclaw.ai/)
- [SimpleClaw](https://www.simpleclaw.com/)
- [UniClaw](https://uniclaw.ai/)
- Every's [Plus One](https://every.to/plus-one)

**优点：**
- 简单，无需设置
- 初创公司每周都在涌现

**缺点：**
- 我总是在某些事情上卡住
- 这些产品会越来越好，我预计这将是大多数人体验 OpenClaw 的方式

---

#### 选项 2：虚拟私有服务器（VPS）💰 最便宜

**供应商：**
- [Railway](https://railway.com/deploy/openclaw)
- [Hostinger](https://www.hostinger.com/vps/docker/openclaw)
- [DigitalOcean](https://www.digitalocean.com/community/tutorials/how-to-run-openclaw)
- [Google Cloud](https://docs.openclaw.ai/install/gcp)
- [Render](https://docs.openclaw.ai/install/render)

**优点：**
- 最便宜的选择
- 快速、强大
- 不需要新硬件

**缺点：**
- 最复杂
- 我没有尝试过这条路，但许多技术人员更喜欢它

---

#### 选项 3：Mac Mini（或旧笔记本）🎮 最有趣

**推荐配置：**
- M4, 16GB, 256GB
- 约 $600

**优点：**
- 简单、强大、紧凑
- 最有趣、最有教育意义、最可爱
- 已经成为一个 meme

**缺点：**
- 最昂贵（至少前期）
- 耗时

**设置清单：**
- 设置新的管理员账户和密码
- [为代理注册 Gmail 地址](https://accounts.google.com/signin)（稍后可以给它只读日历访问权限）
- 确保安装 [Chrome](https://www.google.com/chrome/)（OpenClaw 首选浏览器）
- 抓取键盘、鼠标和显示器进行初始设置（之后不再需要）

## 🚀 安装步骤

### 步骤 1：运行安装脚本

```bash
# 打开终端（Command ⌘ + Space，输入"Terminal"，按 Return）
curl -fsSL https://openclaw.ai/install.sh | bash
```

这将安装你需要的一切，并让你进入 onboarding。

**如果卡住：** 安装 [Claude Code](https://code.claude.com) 和/或 [Codex](https://developers.openai.com/codex/cli) 并寻求帮助！

## 🎭 设置你的代理身份

这是你戴上经理帽子的地方。

就像一个员工，它不能擅长一切，所以为你的 OpenClaw 考虑一个特定的工作。

**角色想法：**
- 个人助理？
- 社交媒体经理？
- 工程实习生？

**从一个想法开始**，你以后总是可以添加更多。

**如果你不确定从哪里开始**，像我一样从个人助理开始。

---

### Onboarding 问题

一旦你的 OpenClaw hatch，它将开始询问你关于你自己和它自己的问题，以构建其定义和运营模型。

**你应该分享：**
- 你的名字
- 你的角色/工作
- 你生活中的常见行政挑战（调度、记住任务、与家人的协调）

---

### 身份文件

所有这些都由代理写下来，并存储在你 onboarding 期间设置的工作空间文件夹中（通常是 `.openclaw/[agent_name]-workspace`）作为 Markdown 文件。

这些文件成为你代理的身份和操作系统。

它们每次 OpenClaw 启动时都被读取，给你的代理做好工作所需的一切。

**查看这些文件很有趣；偶尔，你可能需要进去编辑它们。**

| 文件 | 说明 |
|------|------|
| **AGENTS.md** | OpenClaw 的核心指令和记忆 |
| **SOUL.md** | 你代理的角色、语气和明确边界 |
| **IDENTITY.md** | 你代理的名称、氛围和个人表情符号 |
| **TOOLS.md** | 关于工具以及你的代理应如何使用它们的说明 |
| **USER.md** | 关于你，你 OpenClaw 的人类 |

## 👥 多代理设置

我喜欢 OpenClaw 的灵活性；感觉它几乎可以做任何事情。

**但我发现你不应该尝试让一个代理做所有事情。**

就像我一样，你可以在同一台机器上设置多个 OpenClaw 代理。

```bash
# 运行这个添加新代理
openclaw agents add agent_name
```

你将再次进入 onboarding 以获得全新的代理。

**这个代理可以有：**
- 完全不同的身份
- 工具集
- crons
- 工作空间

**这意味着你可以将它视为完全不同的员工。**

---

### 代理大脑手术

你甚至可以让一个代理生成另一个代理。尝试这样的事情：

```
嘿 Bob，我刚刚设置了 Annie 营销实习生。
将你 SOUL 中关于营销的所有内容、记忆和 crons 转移到她的工作空间，并从你的工作空间中擦除它。
```

**你的 Claws 可以进行大脑手术！**

## 🔗 关键集成

没有集成，OpenClaw 可能感觉像更复杂、更难安装的 Claude Code。

**但有了集成，魔法就发生了。**

---

### 1️⃣ Gog（Google 集成）

**用途：** 使用电子邮件、安排事件、编写文档

**工具：** [gog](https://github.com/nat/gog) — 用于集成 Gmail、Google 日历和 Google Drive 的 CLI 工具

**权限选项：**
- 给你的 Claw 自己的电子邮件地址
- 只读访问你的日历
- 只读访问你的电子邮件
- 如果你勇敢 — 对所有内容的写入访问

**设置方法：** 问 OpenClaw，"我如何给你只读访问我的 Gmail？"

---

### 2️⃣ 网络搜索

**用途：** 研究和查找信息

**选项：**
- Brave API（预加载）
- Exa 搜索（我使用）
- Perplexity
- Firecrawl

---

### 3️⃣ GitHub

**用途：** 编码

**设置：** 给它一个范围狭窄的 Personal Access Token，OpenClaw 成为按需开发人员。

---

### 4️⃣ Linear

**用途：** 分配任务

**设置：** 给它一个 Linear token，这样它就可以分配我工作或传递给我们的团队。

---

### 5️⃣ Obsidian

**用途：** 与代理共享文档

**说明：** 因为 OpenClaw 喜欢编写 Markdown，许多人使用 Obsidian 作为与代理的共享事实来源和协作空间。
