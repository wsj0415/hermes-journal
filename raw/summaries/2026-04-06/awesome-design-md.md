## 🎯 核心概念

### 什么是 DESIGN.md？

**DESIGN.md 是 Google Stitch 引入的新概念：**
- 纯文本设计系统文档
- AI 代理读取它生成一致的 UI
- 只是 markdown 文件
- 无需 Figma 导出、无需 JSON schema、无需特殊工具
- 放到项目根目录，任何 AI 编码代理或 Google Stitch 立即理解 UI 应该如何看起来

**Markdown 是 LLM 阅读最佳的格式，所以无需解析或配置。**

---

| 文件 | 谁读取 | 定义什么 |

## 📋 DESIGN.md 格式（9 个部分）

每个文件遵循 [Stitch DESIGN.md 格式](https://stitch.withgoogle.com/docs/design-md/format/) 并扩展：

| # | 部分 | 捕获内容 |
|----|------|----------|
| **1** | 视觉主题与氛围 | 情绪、密度、设计哲学 |
| **2** | 调色板与角色 | 语义名称 + hex + 功能角色 |
| **3** | 排版规则 | 字体系列、完整层级表 |
| **4** | 组件样式 | 按钮、卡片、输入、导航及状态 |
| **5** | 布局原则 | 间距比例、网格、空白哲学 |
| **6** | 深度与提升 | 阴影系统、表面层级 |
| **7** | Do's and Don'ts | 设计护栏和反模式 |
| **8** | 响应式行为 | 断点、触摸目标、折叠策略 |
| **9** | 代理提示指南 | 快速颜色参考、即用提示词 |

## 🚀 如何使用

### 方式 1：复制 DESIGN.md

```bash
# 1. 复制网站的 DESIGN.md 到你的项目根目录
cp design-md/stripe/DESIGN.md ./DESIGN.md

# 2. 告诉你的 AI 代理使用它
"build me a page that looks like this"
```

**结果：** 获得像素完美 UI，实际匹配设计。

---

### 方式 2：请求新 DESIGN.md

[通过此模板打开 GitHub issue](https://github.com/VoltAgent/awesome-design-md/issues/new?template=design-md-request.yml) 请求为网站生成 DESIGN.md。
