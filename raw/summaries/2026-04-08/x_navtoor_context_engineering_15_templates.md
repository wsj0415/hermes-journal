## 📊 核心洞察

Nav Toor 继上篇"上下文工程杀死提示工程"病毒文章后，这次给出了**15 个可直接复制粘贴的模板**，无需代码、无需 API，5 分钟内让 AI 输出质量大幅提升。

### 核心定义

> **上下文工程** = 设计 AI 模型生成响应前看到的一切  
> 不只是你的提示，还包括：系统指令、记忆、工具、检索知识、动态组装

> **Andrej Karpathy**: LLM 是 CPU，上下文窗口是 RAM，**你是操作系统** — 你决定每个任务加载什么到内存

## 🎯 15 个模板详解

### 第一层：身份层 (Templates 1-3)
**回答**: "我在和谁对话？" — 设置一次，永久生效

#### Template 1: 个人上下文文件 ⭐⭐⭐⭐⭐
**用途**: 消除冷启动问题，让 AI 永久知道你是谁

```
我是 [名字]，[角色] @ [公司/背景]。

我的受众：[客户/读者/团队/高管]。

当前优先级：[列出 2-3 个活跃项目/目标]。

沟通风格：[直接/正式/休闲/技术]。我重视 [简洁/详尽/数据驱动]。

当我说"起草"，我要可编辑的粗稿。当我说"撰写"，我要接近终稿。

默认格式：[要点/短段落/编号列表]。

如不确定，先问我再猜测。绝不编造数据。
```

**为什么有效**: Anthropic 工程团队称系统指令是"控制 Claude 行为的最重要杠杆"

---

#### Template 2: 品牌声音文件 ⭐⭐⭐⭐⭐
**用途**: 防止 AI 听起来像通用聊天机器人，让它听起来像你

```
我的写作声音规则：

语气：[例如：直接自信，无模糊或填充短语]
句子长度：[例如：短。每段最多 2-4 句]
我使用的词：[列出 5-10 个定义你风格的词/短语]
我从不使用的词：[例如："leverage"、"utilize"、"delve"、"I'd be happy to"]
格式：[例如：加粗强调，无斜体，无 emoji 除非指定]
结构：[例如：结论先行，然后用证据支持]

以下是我实际写作的两个示例：
[粘贴你写的一个真实段落]
[粘贴你写的另一个真实段落]

在所有为我写的内容中匹配这个声音。
```

**为什么有效**: 两个真实写作示例胜过千言万语的指令

---

#### Template 3: 工作规则文件 ⭐⭐⭐⭐
**用途**: 阻止 AI 做你讨厌的事

```
每个任务的规则：

1. 开始复杂任务前先问澄清问题。不要假设。
2. 呈现选项时，先给推荐，再给替代方案。
3. 绝不使用占位符文本。每个示例必须具体真实。
4. 如任务需要当前数据，告诉我你无法验证而非编造数字。
5. 默认输出长度：[例如：300-500 字，除非我另指定]。
6. 当我说"更短"，削减 40%。当我说"更长"，扩展 50%。
7. [添加你自己的规则：AI 一直做错你想永久停止的事]。
```

**为什么有效**: 你对 AI 输出的每个抱怨都是一个你还没写的规则

### 第二层：任务层 (Templates 4-6)
**回答**: "你现在具体需要什么？" — 每次新请求使用

#### Template 4: 研究简报 ⭐⭐⭐⭐
**用途**: 防止表面级总结，获取深度研究

```
研究 [主题]。

范围：[我具体需要知道的 — 不是"关于 X 的一切"]
深度：[表面概述/详细分析/全面深入]
来源：[优先原始来源/学术论文/行业报告/最近新闻]
时间框架：[仅过去 6 个月/2 年/所有时间]
输出格式：[要点总结/结构化分节报告/对比表]

对每个声明，包含来源名称和日期。标记任何不确定的内容。

不要包含我已知道的一般背景。我知道 [陈述你已知道的，让 AI 跳过]。
```

**为什么有效**: "不要包含我已知道的一般背景"消除了浪费前三分之一的填充段落

---

#### Template 5: 写作简报 ⭐⭐⭐⭐⭐
**用途**: 任何内容创作任务

```
撰写 [内容类型：邮件/文章/报告/社交帖子]。

受众：[谁会读这个，他们在乎什么]
目标：[读者读后应该想/感觉/做什么]
关键包含点：[列出必须出现的 3-5 件事]
关键排除点：[不应提及什么] ← 最被低估的字段
语气：[参考我的品牌声音文件/或指定：正式、休闲、挑衅、共情]
长度：[具体字数或范围]
结构：[如何组织 — 分节、编号列表、叙事]
参考：[粘贴或描述此任务"好"的示例]
```

**为什么有效**: "关键排除点"是最被低估的字段 — 约束产生专注，专注产生质量

---

#### Template 6: 决策框架 ⭐⭐⭐⭐⭐
**用途**: 让 AI 帮你思考，不只是回答

```
我需要在 [选项 A] 和 [选项 B]（及 [选项 C] 如适用）之间决定。

背景：[描述情况和为何此决定重要]
约束：[预算、时间线、团队规模、技术限制]
我的优先级：[什么最重要 — 速度、成本、质量、风险降低]
我倾向于：[你的当前直觉和原因]

给我：
1. 最重要标准的对比表
2. 每个选项的最强支持论点
3. 每个选项的最强反对论点
4. 你的推荐及具体理由
5. 我可能忽略的

不要中立。采取立场。← 最强大指令
```

**为什么有效**: "不要中立。采取立场。"是决策支持最强大指令 — 没有它 AI 默认"视情况而定"的模糊

### 第三层：控制层 (Templates 7-9)
**回答**: "我如何让 AI 保持轨道？" — 出问题或复杂时使用

#### Template 7: 会话启动 ⭐⭐⭐⭐
**用途**: 任何复杂多步骤会话开始前

```
在我们开始前，这是本次会话的上下文：

项目：[名称和一行描述]
当前状态：[现在进展如何]
上次会话完成：[之前工作总结，如适用]
本次会话目标：[具体交付物或成果]
参考文件：[列出要使用的文档、链接或之前输出]
约束：[时间、字数、格式、范围限制]

确认你理解完整上下文再开始。
```

**为什么有效**: 这是上下文加载 — 在模型生成任何 token 之前前置一切所需

---

#### Template 8: 反幻觉守卫 ⭐⭐⭐⭐⭐
**用途**: 任何涉及事实、数据、声明的任务

```
此任务重要规则：

1. 仅包含你可直接从提供的来源或训练数据验证的信息。
2. 如对特定事实、数字、日期不确定，在旁边用 [UNVERIFIED] 标记。
3. 不编造统计。不捏造引用。不创建虚假引用。
4. 如我问超出你知识截止的事，明确说明而非猜测。
5. 清晰区分"这是事实"和"这是我的分析"。
```

**为什么有效**: [UNVERIFIED] 标签系统是最简单的反幻觉技术 — 给 AI 明确的出口阀，模型在有权限标记不确定时更不可能编造

---

#### Template 9: 上下文重置 ⭐⭐⭐⭐
**用途**: 当对话偏离轨道时

```
停止。让我们重置此对话。

这是我们实际要完成的：[重述原始目标]
哪里出错了：[描述什么偏离轨道]
我现在要你不同地做的：[具体指令]

忽略此消息之上与这些新指令矛盾的一切。从此点重新开始。
```

**为什么有效**: 长对话遭受"上下文腐烂" — 模型注意力漂移到最近消息，丢失原始指令。干净重置胜过 10 条纠正消息

### 第四层：系统层 (Templates 10-12)
**回答**: "我如何构建工作流，不只是对话？" — 专业和开发工作使用

#### Template 10: CLAUDE.md / 项目规则文件 ⭐⭐⭐⭐⭐
**用途**: Claude Code、Cursor 或任何 AI 编码工具

```
- 语言：[例如：TypeScript]
- 框架：[例如：Next.js 15]
- 数据库：[例如：PostgreSQL + Prisma ORM]
- 测试：[例如：Vitest 单元测试，Playwright E2E]
- 使用函数组件。无类。
- 所有函数必须有显式返回类型。
- 错误处理：使用 Result 类型，非 try-catch。
- 文件命名：kebab-case。每文件一个组件。
- 写实现代码前先写测试。
- 标记任何任务完成前运行所有测试。
- 绝不修改当前任务范围外的文件。
- 如对架构决策不确定，实施前先问。
- 不在生产代码中添加 console.log。
- 未经明确批准不安装新依赖。
- 除非任务明确要求，不重构现有代码。
```

**为什么有效**: 此文件在每次 Claude Code 会话开始加载 — 每个指令自动应用于每个任务

---

#### Template 11: 多步骤任务规划器 ⭐⭐⭐⭐⭐
**用途**: 防止 AI 匆忙完成复杂任务

```
我需要你完成一个复杂任务。不要立即开始执行。

首先，创建一个编号计划，包含完成此任务所需的每一步：
- 需要什么信息
- 按什么顺序工作
- 每步交付物看起来如何
- 每步可能出什么错

在执行任何步骤前，呈现计划供我批准。我批准后，一次一步工作，确认完成再进入下一步。

任务：[描述你需要的]
```

**为什么有效**: 这是计划 - 执行 - 验证循环 — AI 最大失败模式是未思考就匆忙输出

---

#### Template 12: 自评估循环 ⭐⭐⭐⭐⭐
**用途**: 让 AI 在展示给你之前检查自己的工作

```
完成此任务后，在呈现给我之前评估你自己的输出。

按这些标准检查：
1. 是否完全解决我列出的每个要求？
2. 有任何事实可疑或未验证的吗？
3. 长度在指定范围内吗？
4. 语气匹配我的品牌声音吗？
5. 有任何无价值的填充句吗？删除它们。
6. 我会舒适地署上我的名字吗？如不，需要什么改变？

如输出任何检查失败，修订后再展示给我。只展示最佳版本。
```

**为什么有效**: 这一个模板可将你的编辑时间减半 — AI 成为自己的第一审稿人，展示第二稿给你

---

### 第五层：自动化层 (Templates 13-15)
**回答**: "我如何让这无需我运行？" — 将一次性工作转为可重复系统

#### Template 13: 每周简报系统 ⭐⭐⭐⭐
**用途**: 每周一运行或设为定期任务

```
使用此结构生成我的每周简报：

1. 优先行动：本周我需要行动的 3 件最重要的事是什么？基于 [我的日历/邮件/项目状态]。

2. 会议准备：列出本周会议。每个包含：谁参加、会议关于什么、我应该准备的一件事。

3. 跟进：列出上周我说要跟进但尚未完成的任何事。

4. 阅读列表：总结过去 7 天 [我的行业/主题领域] 的 3 篇最相关文章或更新。

每部分保持简洁。每部分不超过 5 个要点。以最紧急项开头。
```

**为什么有效**: 这是应用于工作流自动化的上下文工程 — 设计一个重复运行且随上下文文件变好而改进的信息系统

---

#### Template 14: 会议笔记处理器 ⭐⭐⭐⭐⭐
**用途**: 每次会议后从原始笔记提取结构化价值

```
这是我的原始会议笔记：[粘贴或上传笔记]

将这些处理为以下结构：

1. 做出的决定：决定了什么？每个决定列为清晰陈述。
2. 行动项：谁需要做什么，何时截止？格式：[人] — [行动] — [截止日期]
3. 开放问题：什么被提出但未解决？
4. 关键引用：任何值得逐字保留的重要陈述。
5. 下一步：接下来发生什么，下次会议何时？

如笔记中任何信息不清晰，用 [UNCLEAR] 标记而非猜测。
```

**为什么有效**: 原始会议笔记是混乱的 — 这应用"压缩上下文"策略，将大而无结构的输入压缩为仅重要的 token

---

#### Template 15: 技能构建器 ⭐⭐⭐⭐⭐
**用途**: 将任何可重复任务转为永久上下文模板

```
我刚完成一个未来需要重复做的任务。帮我把它转为可复用模板。

任务：[描述你刚做的]
什么有效：[输出的哪些方面好]
我手动修复的：[AI 做错我纠正的]
我下次想要不同的：[下次迭代的改进]

创建一个详细指令模板，我可保存和复用。包含：
- AI 开始前需要知道的确切上下文
- 完成任务的逐步指令
- 质量标准和约束
- 常见要避免的错误（基于我这次必须修复的）
- 输出格式和结构

写它让我可粘贴到任何 AI 工具中，每次获得一致结果。
```

**为什么有效**: 这是元上下文工程 — 用 AI 构建你自己的上下文模板。每个重复任务成为模板，每个模板消除一个你再也不必写的提示
