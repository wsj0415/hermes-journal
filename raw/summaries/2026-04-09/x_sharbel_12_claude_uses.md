## 🎯 完整翻译

### 引言

大多数人像使用**语法更好的搜索引擎**一样使用 Claude。

他们输入问题 → 阅读答案 → 关闭标签继续。

如果是你，你**每次会话都浪费了 90% 的工具功能**。每月付$20的人从未尝试过其中任何一个。

以下是 Claude 真正能做的 12 件事。**按改变输出的程度排序。**

---

## 🔥 12 件 Claude 能做的事

### 1. Steelman 反对你的立场 ⭐⭐⭐⭐⭐

```
"我决定做 X。
我想要你构建每一个理由说明这是错的。"
```

Claude 会构建反对你自己立场的**最佳可能案例**。具体异议。命名的失败模式。你的想法中不工作的确切版本。

**你停止在自己的盲点内做决策。**

大多数人用 Claude 验证他们已经相信的。这是相反的。这是在别人发现之前找到你计划中漏洞的最快方式。

---

### 2. 采访式提问 ⭐⭐⭐⭐⭐

在写任何东西之前，试试这个：

```
"问我问题直到你准确理解我想说什么。
先不要写任何东西。"
```

Claude 会问 **6-10 个有针对性的问题**。你没想过要回答的问题。暴露你认为你知道和实际能表达之间差距的问题。

然后它写。输出听起来像你，不是模板。

**大多数人倾倒模糊提示然后想知道为什么输出感觉通用。采访步骤是你的输出通用的原因。**

---

### 3. 角色扮演读者 ⭐⭐⭐⭐⭐

写这个：

```
"作为 skeptical 的 Series A 投资者阅读这个，
看过 400 份 pitch deck。
告诉我你在哪里失去他们。"
```

Claude 进入那个人的框架。他们的先验。他们的异议。他们的耐心阈值。

你可以为任何观众运行这个。技术招聘经理。首次买家。截止日期的记者。不关心你产品愿景的 CFO。

**大多数人问"这个好吗？"Claude 无法回答不知道对谁好。**

---

### 4. 声音匹配 ⭐⭐⭐⭐⭐

将你的 **5-10 篇写作**粘贴到 Project 中。然后在系统提示中添加一行：

```
"学习这些样本并在每个响应中匹配我的声音。"
```

Claude 提取你的模式。你的句子长度。你的词汇选择。你的节奏。你从未有意识注意到的你做的事情。

**之后的每个输出听起来像你在好日子写的。**

大多数人获得通用 Claude 散文并称"足够接近"。**不够接近。你的声音是你的品牌。**

---

### 5. 决策框架 ⭐⭐⭐⭐⭐

给 Claude 一个你卡住的真实决策。不是假设。一个活的。

告诉它：

```
"我需要在周五前决定 X。
这是我正在权衡的变量 [列出变量]。
为我构建一个带明确标准、权重和我应该问但没问的问题的决策框架。"
```

Claude 会给你一个**结构化矩阵**。排名标准。明确权衡。你当前思维构建的隐藏假设。

**大多数人用 Claude 写作。用它赢的人用它思考。**

---

### 6. 长文档摘要 ⭐⭐⭐⭐

上传 200 页 PDF。合同。市场研究报告。竞争对手白皮书。学术研究。

问：

```
"给我 500 字简报，包含
5 个最操作相关的发现和
3 个我这周应该行动的事情。"
```

你获得信号无噪音。**40 秒内。**

**大多数人要么不读 200 页要么花 3 小时读它们。两者都是错误答案。**

---

### 7. 压力测试计划 ⭐⭐⭐⭐⭐

写出你的上市计划、招聘策略、定价模型。无论你现在赌什么。

然后问：

```
"你是一个聪明的、愤世嫉俗的操作员，看过
50 家公司尝试这个确切方法。
哪里会断裂？
第 9 个月的失败看起来像什么？"
```

Claude 会给你**失败的具体版本**。不是通用风险因素。以你解释出了什么问题结束的**实际事件序列**。

**这是大多数创始人跳过的压力测试因为不舒服。那种不舒服正是它有效的原因。**

---

### 8. 数据分析 ⭐⭐⭐⭐

上传 CSV。电子表格。CRM 或分析仪表板的混乱导出。

问：

```
"找到解释为什么收入在 3 月下降的模式。
向我展示行为不同的细分市场。"
```

Claude 编写 Python、执行它，并交给你输出。表格。图表。**解释数字的数字。**

无 Jupyter notebook。无分析师。无三天周转。

**大多数坐在未分析数据上的人不是在等时间。他们在等 permission 相信答案在里面。它在。**

---

### 9. 运行简报 ⭐⭐⭐⭐⭐

创建 Project。添加名为"Running Brief"或"State of Play"或你脑海中实际称呼的文件。

在每个会话结束时，告诉 Claude：

```
"用我们今天决定的和仍然开放的更新运行简报。"
```

该文件成为**活记录**。每个决策。每个开放问题。每个你做过但尚未测试的假设。

**大多数人在会话间丢失线索。这是线索。**

---

### 10. 困难对话排练 ⭐⭐⭐⭐⭐

告诉 Claude：

```
"扮演我最难的董事会成员。
我需要告诉他们我们 Q2 差 30%。
反驳我说的每件事。"
```

练习对话。被摧毁。找到你没有答案的问题的答案。

然后进行真实对话。

**大多数人走进困难对话准备不足因为排练感觉尴尬。它尴尬。这就是重点。练习中尴尬意味着房间中自信。**

---

### 11. 多观众重写 ⭐⭐⭐⭐

你写一件内容。一个备忘录。一个提案。一个投资者更新。

然后：

```
"为关心实施的技术观众重写这个。
现在为关心风险的非技术高管重写。
现在作为 3  bullet Slack 消息重写。"
```

**一个来源。三个观众。三个格式。每个 30 秒。**

**大多数人从头编写单独版本。那不是写作问题。那是工作流问题。**

---

### 12. 构建可重复框架 ⭐⭐⭐⭐⭐

这是将**10% 更好的人**与**10 倍更快的人**分开的。

不要问 Claude 答案。问 Claude 构建**每次产生答案的流程**。

```
"为我构建一个评估新市场机会的可重复框架。
我想要一个检查清单、评分 rubric 和
我每次问的 5 个问题。"
```

你**在未来 3 年使用该框架**。你通过它运行每个新决策。你把它交给你的团队。

**大多数人得到鱼。你可以问钓鱼竿。几乎没人做。**

### 对 KilroyContentBot 的启示

#### 短期（立即可用）
1. **采访式提问**
   - 在内容创作前先问用户 6-10 个问题
   - 澄清需求、受众、语气、长度

2. **角色扮演**
   - 分析内容时扮演 skeptical 投资者/技术专家
   - 找出弱点和改进点

3. **运行简报**
   - 每次会话后更新 `memory/running-brief.md`
   - 追踪决策和开放问题

#### 中期（1-2 周）
1. **声音匹配**
   - 收集用户写作样本到 `voice-samples/`
   - 在系统提示中添加声音匹配指令

2. **决策框架**
   - 为内容选题构建评分 rubric
   - 追踪每个选题的决策过程

3. **困难对话排练**
   - 模拟用户与投资者/客户对话
   - 准备 Q&A 文档

#### 长期（1 月）
1. **可重复框架**
   - 将 X 分析工作流编码为框架
   - 内容创作 SOP
   - 多平台改编流程

2. **与 Wiki 集成**
   - 12 用法的输出归档到 wiki/
   - 决策 → 决策日志
   - 框架 → 可复用模板
