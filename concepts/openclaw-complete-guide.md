## 什么是 OpenClaw

**OpenClaw 是一个开源个人 AI 助手**，比试过的任何东西都更强大、更自主、更有趣。

### 核心特性

| 特性 | 说明 |
|------|------|
| **消息平台** | 在你已经使用的平台上轻松消息它：WhatsApp、Telegram、Slack 等 |
| **电脑控制** | 可以控制你的电脑，像你自己一样完成任务 |
| **自主工作** | 按照自己的时间表工作，甚至通宵 |
| **始终在线** | 本地运行，可以构建自己的新技能 |
| **一次教学** | 教一次，它处理其余 |

### 实际示例

我可以给我的 OpenClaw 发这样的消息：

> "让我们确保我们的网站始终有最新的理由说明我们为什么比竞争对手更好。"

**它会**：
1. 使用网络搜索、我们的 GitHub 仓库和公共 API 找到所需信息
2. 提交 PR 与更新
3. 每周根据新功能或市场新闻刷新这些页面

**一条短信变成一个始终在线的代理。**

## 关键 OpenClaw 概念

| 概念 | 说明 |
|------|------|
| **本地网关** | 接收消息的通用收件箱，可以从任何渠道接收指令（终端、Telegram、WhatsApp 等） |
| **代理** | 在网关后面，有自己的身份、工具和工作空间 |
| **Cron 作业 + 心跳** | 代理按照预定的 cron 作业工作，每 30 分钟检查一次心跳 |
| **技能和工具** | 可以使用（并自安装！）技能、API 和 CLI 与系统和外部世界交互 |
| **部署** | 部署在拥有的机器上，如 Mac Mini 或云端的 VPS |

## 安装选项

### ⚠️ 重要警告

**不要**在积极使用的工作或个人电脑上安装 OpenClaw。这非常危险。

OpenClaw technically 可以访问它运行电脑上所有文件，无论多么小心，你都不想冒删除一切或将个人文件发送给不可信角色的风险。

---

### 三个安全选项

#### 选项 1：托管版本 ⭐ 最简单

**供应商**：
- [StartClaw](https://startclaw.com/)
- [MyClaw](https://myclaw.ai/)
- [SimpleClaw](https://www.simpleclaw.com/)
- [UniClaw](https://uniclaw.ai/)
- Every's [Plus One](https://every.to/plus-one)

**优点**：
- 简单，无需设置
- 初创公司每周都在涌现

**缺点**：
- 我总是在某些事情上卡住
- 这些产品会越来越好，我预计这将是大多数人体验 OpenClaw 的方式

---

#### 选项 2：虚拟私有服务器（VPS）💰 最便宜

**供应商**：
- [Railway](https://railway.com/deploy/openclaw)
- [Hostinger](https://www.hostinger.com/vps/docker/openclaw)
- [DigitalOcean](https://www.digitalocean.com/community/tutorials/how-to-run-openclaw)
- [Google Cloud](https://docs.openclaw.ai/install/gcp)
- [Render](https://docs.openclaw.ai/install/render)

**成本**：$5-20/月

**优点**：
- 便宜
- 隔离环境
- 24/7 运行

**缺点**：
- 需要一些技术知识
- 调试可能复杂

---

#### 选项 3：本地机器 ⭐ 最推荐

**硬件**：
- Mac Mini M2/M3（推荐）
- 任何有足够 RAM 的电脑（16GB+）
- 树莓派 5（用于轻量使用）

**优点**：
- 完全控制
- 无月费
- 数据隐私
- 学习最多

**缺点**：
- 前期硬件成本
- 需要设置和维护

## 第一个代理

### 步骤 1：选择代理身份

**示例**：
- **Polly** — 个人助理（日历、邮件、调度）
- **Dev** — 开发者代理（代码、PR、部署）
- **Sales** — 销售代理（潜在客户跟进、邮件）
- **Research** — 研究代理（网络搜索、报告）

### 步骤 2：配置工具和权限

**基本工具**：
- 日历访问（Google Calendar）
- 邮件访问（Gmail）
- GitHub 访问
- 网络搜索（Tavily/Google）

**权限级别**：
- **只读** — 安全，仅查看
- **写入** — 可以创建/编辑
- **完全** — 可以删除/部署（谨慎！）

### 步骤 3：定义第一个任务

**简单任务开始**：
```
"每天早上 8 点，检查我的日历并发送今日议程到 Telegram"
```

**中等任务**：
```
"监控我们的 GitHub 仓库，当有新的 issue 时，分类并分配给合适的团队成员"
```

**复杂任务**：
```
"运营我们的内容营销：每周写 3 篇博客，发布到网站，推广到社交媒体"
```

## 实际使用案例

### 案例 1：运营业务

**代理团队**：
- **Sales Agent** — 处理潜在客户，安排演示
- **Support Agent** — 回答常见问题，创建工单
- **Content Agent** — 写博客，管理社交媒体
- **Dev Agent** — 修复 bug，部署更新

**结果**：业务几乎自主运行，创始人只需每周审查。

---

### 案例 2：个人生活管理

**代理**：
- **Polly** — 个人助理
  - 管理日历
  - 安排约会
  - 提醒重要日期（生日、周年）
  - 订购日用品

**结果**：不再错过约会，不再忘记重要日期。

---

### 案例 3：内容创作

**工作流**：
```
1. Research Agent 研究趋势主题
2. Writer Agent 起草内容
3. Editor Agent 编辑和格式化
4. Publisher Agent 发布到各平台
5. Analytics Agent 追踪表现
```

**结果**：每周产出 5-10 篇内容，无需手动工作。

## 培训和教学

### 如何教你的代理

**方法 1：示例驱动**

```
"这是我喜欢的工作方式：[示例]
以后类似任务请按这个模式处理。"
```

**方法 2：反馈循环**

```
"这个输出不错，但下次请：
- 更简洁
- 包含更多数据
- 使用项目符号"
```

**方法 3：技能安装**

```
"安装这个技能：[技能链接]
它将帮助你更好地处理 [任务类型]。"
```

### 常见陷阱

| 陷阱 | 解决方案 |
|------|----------|
| **代理太激进** | 降低权限级别，添加审批步骤 |
| **代理太保守** | 增加权限，提供明确示例 |
| **代理忘记事情** | 启用持久记忆，定期审查 |
| **代理产生幻觉** | 添加验证步骤，要求引用来源 |
| **代理成本过高** | 设置预算限制，优化提示词 |

## 安全和治理

### 安全最佳实践

1. **从隔离环境开始** — 不要直接在主电脑上运行
2. **逐步增加权限** — 从只读开始，逐步提升
3. **设置预算限制** — 防止意外高成本
4. **启用审批流程** — 关键操作需要确认
5. **定期审查日志** — 检查代理行为
6. **备份重要数据** — 防止意外删除

### 治理框架

| 操作类型 | 审批要求 |
|----------|----------|
| 读取文件 | 无需审批 |
| 创建文件 | 无需审批 |
| 编辑文件 | 无需审批 |
| 删除文件 | 需要审批 |
| 部署代码 | 需要审批 |
| 发送外部消息 | 需要审批 |
| 花费 >$10 | 需要审批 |

## 关键洞察

1. **从简单开始** — 第一个代理做单一任务
2. **安全优先** — 隔离环境，逐步授权
3. **培训是关键** — 代理需要明确指导
4. **监控和审查** — 定期检查代理行为
5. **复合效应** — 多个代理协同工作产生巨大价值
