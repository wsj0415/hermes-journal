## 核心问题

**Claude 总是附和你。每次都是。你不能信任它。**

### Claude 的附和性

- 问它"我应该推出这个产品吗？" — 它会找到 5 个理由说应该
- 问它"这个产品是坏主意吗？" — 它会找到 5 个理由说是

**同样的产品，不同的框架，完全相反的答案。**

### 问题根源

你得到的答案被以下因素塑造：
- ✅ 你的假设
- ✅ 你的框架
- ✅ 你的情感倾向

Claude 捕捉到所有这些，然后告诉你你想听的。

**这对写邮件没问题。但对做决策很危险。**

---

## 解决方案：委员会方法

委员会通过强制 5 种不同的思考方式来看待同一个问题来打破这个循环。

**它们不能都附和你，因为它们不是都从你的角度看问题。**

---

## 来源：Andrej Karpathy 的 LLM 委员会

### 原始概念

**Andrej Karpathy**（OpenAI 联合创始人 + 前特斯拉 AI 主管）构建了 LLM 委员会。

**核心思路**：

1. **投票** — 用同样的问题询问多个模型
2. **同行评审** — 让它们匿名互相审查回答
3. **综合** — 主席模型阅读所有内容并综合最终答案

**Karpathy 版本**：GPT、Claude、Gemini 等模型互相对抗

**Ole 版本**：完全在 Claude 内部重建，使用具有不同思考风格的子代理，而非不同模型

## 5 个顾问角色

### 1️⃣ The Contrarian（反对者）

**职责**：寻找会失败的地方

**思维方式**：
- 假设你的想法有致命缺陷，并试图找到它
- 如果一切看起来稳固，挖得更深

**捕获的盲点**：
> "这听起来很棒，但你想过...吗？" — 当你兴奋时会忽略的漏洞

---

### 2️⃣ The First Principles Thinker（第一性原理思考者）

**职责**：忽略你的问题，问你真正想解决什么

**思维方式**：
- 剥离假设
- 从头重建问题

**捕获的盲点**：
> "你完全在优化错误的变量" — 这比你想象的更常见

---

### 3️⃣ The Expansionist（扩张者）

**职责**：寻找你错过的上升空间

**思维方式**：
- 什么可以更大？
- 你的问题旁边有什么相邻机会你还没注意到？

**捕获的盲点**：
> "你想得太小了"

---

### 4️⃣ The Outsider（局外人）

**职责**：对你、你的领域或你的历史零背景

**思维方式**：
- 纯粹基于眼前的内容回应
- 像完全陌生的人一样思考

**捕获的盲点**：
> 知识的诅咒 — 对你明显但对客户完全不可见的东西

---

### 5️⃣ The Executor（执行者）

**职责**：只关心一件事：周一早上你做什么？

**思维方式**：
- 如果想法听起来很棒但没有清晰的第一步，执行者会说出来

**捕获的盲点**：
> 很棒的计划但没有实际执行路径 — 这是大多数计划的问题

## 工作流程

### Step 1: 提交问题

```
用户向委员会提交问题
```

### Step 2: 5 个顾问独立回答

```
每个顾问从自己的角度分析问题
产生独立的回答和建议
```

### Step 3: 匿名同行评审

```
每个顾问阅读其他 4 个回答
匿名提出批评和补充
```

### Step 4: 主席综合

```
主席模型阅读所有内容
综合最终裁决
标记共识和分歧点
```

### Step 5: 交付最终答案

```
用户收到：
- 5 个独立观点
- 同行评审意见
- 主席综合裁决
- 共识和分歧点
```

## 使用场景

### 场景 1：产品决策

**问题**："我应该推出这个产品吗？"

**传统方式**：
```
Claude 找到 5 个理由说应该
```

**委员会方式**：
```
反对者：找到 3 个致命风险
第一性原理：你解决的是真正的问题吗？
扩张者：可以扩展到相邻市场
局外人：这个价值主张清晰吗？
执行者：第一步是什么？
主席：综合建议，标记风险
```

---

### 场景 2：职业选择

**问题**："我应该换工作吗？"

**传统方式**：
```
Claude 附和你的倾向
```

**委员会方式**：
```
反对者：新工作的风险
第一性原理：你真正想要什么？
扩张者：有其他机会吗？
局外人：客观对比两份工作
执行者：如何平稳过渡？
主席：综合建议
```

---

### 场景 3：技术选型

**问题**："应该用 React 还是 Vue？"

**传统方式**：
```
Claude 给出平衡但无用的回答
```

**委员会方式**：
```
反对者：每个框架的陷阱
第一性原理：你真正需要什么？
扩张者：未来扩展考虑
局外人：团队技能匹配
执行者：迁移路径
主席：基于具体情况的推荐
```

## 实施建议

### 方式 1：使用现有技能

如果有现成的 LLM Council 技能，直接安装使用。

### 方式 2：手动模拟

```
1. 向 Claude 提交问题，要求扮演 5 个角色
2. 让 Claude 依次扮演每个顾问
3. 最后让 Claude 综合裁决
```

**提示词模板**：
```
请扮演 5 个顾问角色，为我的问题提供不同视角：

1. 反对者：寻找会失败的地方
2. 第一性原理思考者：剥离假设，从头重建
3. 扩张者：寻找错过的上升空间
4. 局外人：零背景，纯粹基于内容
5. 执行者：只关心周一早上做什么

问题：[你的问题]

请依次扮演每个角色，最后综合一个裁决。
```

## 关键洞察

1. **附和是默认行为** — LLM 倾向于告诉你你想听的
2. **多样性产生真理** — 不同视角揭示盲点
3. **同行评审是关键** — 让顾问互相审查
4. **综合优于单一** — 主席综合比单一回答更可靠
5. **角色定义清晰** — 每个顾问有明确职责
